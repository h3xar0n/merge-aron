README Markdown Editor

READMEs are essential for developers to share their work with others, but documentation is often overlooked if it is perceived as a hassel to create.

To encourage more engagement in writing strong, useful READMEs, we can include prompts for ideas about what to include. We can also show the user what their README will actually look like before they commit.

The Light Site

With a great text-editor like Atom and many programmers preferring a dark-style environment to read code, it seems that GitHub would greatly benefit from offering an alternative to the standard white.

White and blue may be good for meeting the maximum accessibility standards, but it doesn't have to be the only option.

The Dark Site


Some forms of colorblindness benefit more from dark backgrounds because syntax highlighting is more distinct. It is also easier to read at night or after staring at white pages all day long.

When Google Chrome's developer tools introduced a dark theme, people went wild. I created this by simply altering the CSS in dev tools, and it could easily be implemented with a switch at the top of the page or in settings. Using JavaScript, we could alter the classes of elements across the site to use dark backgrounds and pastel text.

A tiny feature like this could bring a wave of new interest and engagement.
